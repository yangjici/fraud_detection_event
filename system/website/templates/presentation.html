Fraud Detection Case Study - Group 3

Process Flow

Model Development:
After some initial time in EDA, we split model development evenly amongst the team. This included identifying relationships between columns, small data engineering tasks, and hunches regarding potential relevancy. Each team member took a stab at building a different model; a Random Forrest Classifier gave us the best results and also fit our training/prediction tradeoff parameters.
Web App Development:
Architecting the web app involved two separate threads - one to run the web server and one to ping for new data points and predict. Rather than create a threading system, we opted to run each in its own python environment. We then built the flask front end with javascript additions and set up the MongoDB predict processes.

Preprocessing

Data cleaning steps to select a subset of columns, including:
- imputing data
- type conversions
- adding new features

Assessment of Model

- Accuracy
- Precision
- Recall

Validation and Testing

"The Standards":
